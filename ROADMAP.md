# Roadmap

This is a living view of where **agent-readme.md** is headed.  The convention is a draft (v0.1), so priorities will shift with feedback.  For discussion, use [GitHub Discussions](https://github.com/HannahVernon/agent-readme-md/discussions); for tracked work, see [Issues](https://github.com/HannahVernon/agent-readme-md/issues).

## Now

- Gather feedback on the draft v0.1 spec.
- Stand up the community surface: Discussions, Issues, and this roadmap.
- Promote the convention where coding-agent practitioners gather.

## Next

- Incorporate spec feedback from the community.
- Add templates for more project shapes (monorepo, microservices, a data pipeline with destructive operations, a governed open-source library).
- Add inline discussion to the site pages so feedback can happen in context.

## Later

- A dynamic, self-hosted endpoint badge that reflects real compliance, backed by a validator.
- A simple validator/linter (CLI or GitHub Action) that checks an `AGENT-README.md` for well-formedness and completeness.
- A directory of projects that adopt the convention.
- A versioned spec with a published changelog.

## Done

- Draft v0.1 of the specification.
- The website: homepage, spec, comparison with AGENTS.md, and templates.
- An optional flat metadata header (not YAML).
- A self-hosted adoption badge and project mark.
